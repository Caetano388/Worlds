---- elements.json format guide ----

-- File Structure --

{
  "elements": [ -- list of elements --
    {
      "id":                  -- (required) Unique element identifier, if more than one  definition share ids, only the last one loaded will be used
      "name":                -- (required) name of element (in plural form, see note #3)
      "adjectives":          -- (required) list of applicable adjectives, separated by commas
      "constraints":         -- (required) list of application constraints, separated by commas (see note #4)
      "associationStrings":  -- (required) list of sentence association strings, separated by commas (see note #5)
    },
    ... -- additional elements --
  ]
}

-- Notes --
1. Only one elements.json file can be used per world. If more than one elements.json file is present across multiple active mod files, only the last one being loaded will be used.
2. Remove any trailing commas or the file won't be parsed
3. The name string should be written in parseable plural form. Examples: "stone:s", "grass:es". Refer to language_modding_guide.txt for more details on parseable plural names
4. Each constraint within the list must be encased in single quotes.
  Types of valid constraints:
  - 'altitude_above', Example: 'altitude_above:0'
  - 'altitude_below', Example: 'altitude_below:0'
  - 'rainfall_above', Example: 'rainfall_above:675'
  - 'rainfall_below', Example: 'rainfall_below:1775'
  - 'temperature_above', Example: 'temperature_above:-15'
  - 'temperature_below', Example: 'temperature_below:10'
  - 'no_attribute', Example: 'no_attribute:Rainforest,Jungle,Taiga,Forest'
  - 'any_attribute', Example: 'any_attribute:Desert,Delta,Peninsula,Island,Coast'
  - 'any_attribute', Example: 'any_attribute:Grassland,Tundra,Desert'
  - 'main_biome', Example: 'main_biome:Forest,Taiga,Tundra,Rainforest'
5. Each association string within the list must be encased in single quotes.
  Associations are used along with the element and one or more adjectives to build noun phrases.
  They are composed of three parts: '<association word>,<relations>,<form>'
    - <association word>: an action noun or attributive verb. Refer to language_modding_guide.txt for more details on using association words
    - <relations>: one or more prepositions (separated by pipes, '|') that can link the association word with the element
    - <forms>: one or more forms (separated by pipes, '|') the element can take as part of the noun phrase. They can be one of the following:
      -- 'ns': name singular. Examples: "the stone", "mount"
      -- 'ds': definite singular. Examples: "the mount", "stone"
      -- 'is': indefinite singular. Examples: "a stone", "an arch"
      -- 'dp': definite plural. Examples: "the stones", "mounts"
      -- 'ip': indefinite plural. Examples: "stones", "archs"
      -- 'u': uncountable. Examples: "water", "air"
  Example association strings:
    '[nrv]throw:er,of,ip|ns'
    '[iv(bear,ts,past)]born,between,ip'
    '[nrv]break:er,of,ip|ns'
    '[iv(bear,ts,past)]born,under,ns'
    '[nrv]cut:ter,of,u'
